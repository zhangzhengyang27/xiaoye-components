# 前台组件库建设计划

## 架构决策

### 四包定位

| 包 | 定位 | 新/已有 |
|---|---|---|
| `xiaoye-primitives` | 基础设施层：composables + utils + theme 基础 | **新建** |
| `xiaoye-components` | 通用基础组件（62 个） | 已有，**重构依赖** |
| `xiaoye-pro-components` | 中后台增强组件（20+ 个） | 已有 |
| `xiaoye-ui` | 前台产品组件库 | **新建** |

### 依赖关系图（重构后）

```
xiaoye-primitives          xiaoye-components    xiaoye-ui
┌─────────────────┐        ┌──────────────┐    ┌──────────────┐
│ composables/    │ ←────── │ 62 components │    │ front-components │
│ utils/          │        └──────┬───────┘    │ headless/     │
│ theme/          │               │            └──────┬───────┘
└────────┬────────┘               │                   │
         │                        │                   │
         └──────────┬─────────────┘                   │
                    │                               │
                    ↓                               ↓
              xiaoye-pro-components (依赖 xiaoye-components)
```

### `xiaoye-primitives` 的组成

从现有 monorepo 中提取，组成独立的基础设施包：

| 来源 | 内容 |
|------|------|
| `packages/composables/` | 13 个 composables（`useFocusTrap`、`useListNavigation`、`useDismissibleLayer` 等） |
| `packages/utils/` | 工具函数（`packages/utils/index.ts`） |
| `packages/theme/` | CSS 入口 + `reset.css` + `tokens.css` + `base.css` + `shared/display-value.css` |

### `xiaoye-ui` 包含的内容

`xiaoye-ui` 由两部分组成：

1. **前台专属组件** — `xiaoye-components` 和 `xiaoye-pro-components` 都不覆盖的业务组件
2. **Headless 风格组合 API** — 参照 Headless UI Vue 的交互模式，对基础组件进行组合式 API 增强

### Headless UI Vue 10 个组件的覆盖策略

| Headless UI 组件 | 对应基础组件 | xiaoye-ui 中的落位 |
|---|---|---|
| `Dialog` | `packages/components/dialog` | 拆分为 Dialog / DialogPanel / DialogTitle / DialogDescription 组合 + TransitionRoot / TransitionChild |
| `Menu` | `packages/components/dropdown` | 拆分为 Menu / MenuButton / MenuItems / MenuItem 组合 |
| `Listbox` | `packages/components/select` | 新增 Listbox / ListboxButton / ListboxOptions / ListboxOption 组合 |
| `Combobox` | `packages/components/auto-complete` | 基于 Listbox 增强，新增 Combobox / ComboboxInput / ComboboxOptions 组合 |
| `Switch` | `packages/components/switch` | 新增 SwitchGroup / SwitchLabel 封装 |
| `Disclosure` | `packages/components/collapse` | 拆分为 Disclosure / DisclosureButton / DisclosurePanel 组合 |
| `Radio Group` | `packages/components/radio` | 新增 RadioGroup / Radio 组合（已有单文件，补 Group 封装） |
| `Tabs` | `packages/components/tabs` | 参照 Headless UI Tab 模式增强交互 |
| `Popover` | `packages/components/popover` | 拆分为 Popover / PopoverButton / PopoverPanel / PopoverHeading 组合 |
| `Transition` | Vue 内置 `<Transition>` | 新增 TransitionRoot / TransitionChild 封装，支持多层动画协调 |

- dtsInclude：包含 `src/headless/` 和 `src/front-components/`

### 1.5 更新 `packages/components` 的依赖

将 `packages/components/` 下所有源文件中的：

```ts
import { ... } from "@xiaoye/composables";
```

批量替换为：

```ts
import { ... } from "@xiaoye/primitives";
```

同样处理 `packages/pro-components/` 下的源码。

### 1.6 重构 `packages/theme` 样式入口

修改 `packages/theme/index.css`：

- 移除已移走的 `@import "./src/reset.css"`、`@import "./src/tokens.css"`、`@import "./src/base.css"`、`@import "./src/shared/display-value.css"`
- 在文件开头新增：`@import "xiaoye-primitives/style.css";`

### 1.7 更新 `package.json` scripts

根目录 `package.json` 调整 `build` 脚本顺序：

```json
"build:primitives": "vite build -c vite.primitives.config.ts",

```

## 阶段二：前台专属组件开发

### 2.1 ProductCard（商品卡片）

**参照 Headless UI 思路**：Headless UI 的 Menu/Listbox 的 slot props 模式（`active`/`disabled` 状态透出）可以借鉴到卡片交互上。

**功能范围**：
- 图片区：主图 + 缩略图列表（多图时支持切换）
- 信息区：标题、描述、价格（划线价）、标签
- 操作区：购买按钮、收藏按钮
- 状态：hover 高亮、选中态、售罄/下架态
- 尺寸：`sm` / `md` / `lg`
- 交互：键盘可聚焦、悬停显示操作按钮

**Props 设计**：

```ts
interface ProductCardProps {
  image?: string | string[];
  title?: string;
  description?: string;
  price?: number | string;
  originalPrice?: number | string;
  tags?: Array<{ text: string; type?: 'primary' | 'success' | 'warning' | 'danger' }>;
  stock?: number; // 库存，0 时显示售罄
  status?: 'online' | 'offline' | 'soldout';
  size?: 'sm' | 'md' | 'lg';
  horizontal?: boolean; // 横向布局（移动端/列表页场景）
}
```

**插槽**：
- `image` — 自定义图片区
- `header` — 覆盖标签区
- `footer` — 覆盖操作按钮区

**实现依赖**：复用 `packages/components` 的 `image`、`badge`、`tag`、`button`

### 2.2 MarketingModal（营销弹层）

**参照 Headless UI 思路**：Dialog 组件的 `TransitionRoot` + `TransitionChild` 模式，实现 overlay / panel / 内容的分层过渡动画。

**功能范围**：
- 类型：`coupon`（优惠券）、`flash-sale`（限时抢购）、`promotion`（满减活动）、`custom`（自定义）
- 背景遮罩：支持模糊背景、图片背景
- 内容区：自定义插槽，不限制内部结构
- 动画：多层过渡（遮罩淡入 → 面板缩放 → 内容淡入）
- 关闭方式：点击遮罩、按 ESC、点击关闭按钮、倒计时自动关闭
- 变体：`centered`（居中）、`bottom-sheet`（底部滑出，移动端友好）

**Props 设计**：

```ts
interface MarketingModalProps {
  open: boolean;
  type?: 'coupon' | 'flash-sale' | 'promotion' | 'custom';
  title?: string;
  backdropBlur?: boolean; // 遮罩模糊效果
  countdown?: number; // 倒计时秒数，为 0 不显示
  autoClose?: number; // 自动关闭倒计时
  variant?: 'centered' | 'bottom-sheet';
}
```

**实现依赖**：复用 `packages/components` 的 `dialog`、`transition`

### 2.4 SkuSelector（SKU 选择器）

**参照 Headless UI 思路**：Listbox 的键盘导航 + 选中状态管理 + disabled 状态模式。

**功能范围**：

interface SkuSelectorProps {

## 阶段三：交互模式增强（参照 Headless UI Vue 全部 10 个组件）

### 3.0 TransitionRoot / TransitionChild（过渡基础设施）

**参照 Headless UI**：`TransitionRoot` 和 `TransitionChild` 组件，用于协调多层嵌套元素的过渡动画。

**现有状态**：当前 Dialog / Dropdown 等组件各自处理过渡动画，逻辑分散。

**目标**：提供统一的过渡组件，支持：
- `TransitionRoot`：根容器，控制 `show` 状态
- `TransitionChild`：子容器，支持 enter / enter-start / enter-end / leave / leave-start / leave-end 六种状态类
- 与 Vue 内置 `<Transition>` 和 `<TransitionGroup>` 互补，专注于"协调多个元素的时序过渡"

**Props**：

```ts
interface TransitionChildProps {
  show?: boolean;
  enter?: string;         // enter 状态的类名
  enterFrom?: string;    // enter 开始时的类名
  enterTo?: string;      // enter 结束时的类名
  leave?: string;        // leave 状态的类名
  leaveFrom?: string;    // leave 开始时的类名
  leaveTo?: string;      // leave 结束时的类名
  enterInitiallyVisible?: boolean;
}
```

**落位**：`packages/xiaoye-ui/src/headless/transition/`

### 3.1 Dialog 分层组合

**参照 Headless UI**：`Dialog` / `DialogPanel` / `DialogTitle` / `DialogDescription` 分层组合，`TransitionRoot` + `TransitionChild` 协调动画。

**现有状态**：`packages/components/dialog` 是单文件组件，内置遮罩 + 面板 + 过渡逻辑。

**目标**：提供 Headless 风格的分层组合 API，与现有 `XyDialog` 并存：

```vue
<template>
  <TransitionRoot :show="isOpen">
    <!-- 遮罩层 -->
    <TransitionChild
      enter="duration-300 ease-out"
      enter-from="opacity-0"
      enter-to="opacity-100"
      leave="duration-200 ease-in"
      leave-from="opacity-100"
      leave-to="opacity-0"
    >
      <div class="fixed inset-0 bg-black/30" />
    </TransitionChild>

    <!-- 面板层 -->
    <TransitionChild
      enter="duration-300 ease-out"
      enter-from="opacity-0 scale-95"
      enter-to="opacity-100 scale-100"
      leave="duration-200 ease-in"
      leave-from="opacity-100 scale-100"
      leave-to="opacity-0 scale-95"
    >
      <DialogPanel class="xy-dialog-panel">
        <DialogTitle>Title</DialogTitle>
        <DialogDescription>Description</DialogDescription>
        <slot />
      </DialogPanel>
    </TransitionChild>
  </TransitionRoot>
</template>
```

**Props**：
- `Dialog`：提供 `open`、`initialFocus`、自动 Portal 渲染、scroll lock、inert 状态管理
- `DialogPanel`：提供 focus trap、点击外部关闭
- `DialogTitle`：设置 `aria-labelledby`
- `DialogDescription`：设置 `aria-describedby`

**落位**：`packages/xiaoye-ui/src/headless/dialog/`

---

### 3.2 Menu 分层组合

**参照 Headless UI**：`Menu` / `MenuButton` / `MenuItems` / `MenuItem` 分层组合，slot props 透出 `active` / `disabled` / `close`。

**现有状态**：`packages/components/dropdown` 是单文件组件，有 items 数据驱动模式。

**目标**：提供 Headless 风格的自由插槽组合 API：

```vue
<template>
  <Menu>
    <MenuButton class="xy-menu-btn">Options</MenuButton>
    <MenuItems>
      <MenuItem v-slot="{ active, disabled, close }">
        <button :class="{ 'bg-blue-500': active }" @click="close">
          Account settings
        </button>
      </MenuItem>
    </MenuItems>
  </Menu>
</template>
```

**Props**：
- `Menu`：提供 `open`/`close` 状态管理，键盘导航（方向键 / Enter / Esc）
- `MenuButton`：触发器，接受 `as` prop 渲染为任意元素
- `MenuItems`：浮层面板，接受 `static` / `unmount` prop
- `MenuItem`：`disabled` prop，slot props 提供 `active` / `disabled` / `close`

**落位**：`packages/xiaoye-ui/src/headless/menu/`

### 3.3 Listbox 分层组合

**参照 Headless UI**：`Listbox` / `ListboxButton` / `ListboxOptions` / `ListboxOption` 分层组合。

**现有状态**：`packages/components/select` 是单文件组件，有完整的单选/多选模式。

**目标**：提供 Headless 风格的列表框组合 API，专注于无障碍和键盘导航：

```vue
<template>
  <Listbox v-model="selected">
    <ListboxButton class="xy-listbox-btn">{{ selected?.label }}</ListboxButton>
    <ListboxOptions>
      <ListboxOption
        v-for="person in people"
        :key="person.id"
        :value="person"
        v-slot="{ active, selected, disabled }"
      >
        <li :class="{ 'bg-blue-100': active, 'opacity-50': disabled }">
          {{ person.name }}
          <span v-if="selected">*</span>
        </li>
      </ListboxOption>
    </ListboxOptions>
  </Listbox>
</template>
```

**Props**：
- `Listbox`：提供 `modelValue`、`open` 状态、键盘导航
- `ListboxButton`：触发器按钮
- `ListboxOptions`：下拉列表容器
- `ListboxOption`：`value` prop、slot props：`active` / `selected` / `disabled`

**落位**：`packages/xiaoye-ui/src/headless/listbox/`

### 3.4 Combobox 分层组合

**参照 Headless UI**：`Combobox` / `ComboboxInput` / `ComboboxOptions` / `ComboboxOption` 分层组合，是 Listbox 的带搜索变体。

## 执行顺序

1. **阶段一（脚手架）** — 最重要，分三步走：
   - 1.1 创建 `xiaoye-primitives` 包目录、`package.json`、`vite.primitives.config.ts`、入口文件
   - 1.2 移动 `packages/composables`、`packages/utils`、`packages/theme/src/` 的基础文件到 `xiaoye-primitives`
   - 1.3 重构 `xiaoye-components` 和 `xiaoye-pro-components` 的依赖，更新 `@xiaoye/composables` → `@xiaoye/primitives`
   - 1.4 创建 `xiaoye-ui` 包脚手架
2. **阶段二（5 个前台专属组件）** — 按优先级实现，每个组件独立可测试
3. **阶段三（10 个 Headless 组合 API）** — 与阶段二并行，按依赖顺序实现：
   - 先做 TransitionRoot/TransitionChild（其他组件依赖它做动画）
   - 再做 Dialog（最核心，依赖 Transition）
   - 然后 Menu、Listbox、Combobox、SwitchGroup、Disclosure、RadioGroup、Tabs、Popover
4. **阶段四（样式适配）** — 前台层 CSS 变量覆盖，在前两个阶段之间/之后按需调整
5. **阶段五（文档与测试）** — 与组件开发同步进行
6. **阶段六（入口注册）** — 最后收口，确保一致性
